# built for the babysitting problem

You cannot watch a whole fleet at once. Workspacer keeps the ambient state in front of you so you only context-switch when it pays off.

ambient_awareness

Know who needs you

A status dot per agent and a live "N need you / N working" count. Notifications and a taskbar flash fire on needs-input, and stay quiet for the agent you are already watching.

# one workspace per agent

Each agent keeps its own tabs and panes in the daemon, separate from the window. ls the pane types:

terminala real xterm and PTY shell scoped to the agent cwd
browserembedded webview with nav, bookmarks, app mode, themed to match
editorcodemirror with find and replace, vim mode, folding, file tree
reviewgit status, diff, stage, commit, and push in the app
notesper-agent markdown scratchpad that saves with the session
libraryreusable prompts, skills, and agents with variable templating
analyticscost and token totals by project and model, plus recent sessions
overviewcross-agent stats, a rate-limit card, favourite dirs to spawn from
askturn a question into a supervisor agent that drives the rest
pluginsinstall, remove, and health-check sidecars that bring their own panes
# your layout, your way
  • Tabs, spatial, or stacked. Switch layouts without remounting a single live pane, so nothing flickers or resets.
  • The Fleet Deck is a radar of every live agent, on top of whatever view you are in.
  • Command palette, default and vim keybindings with leader chords, and around 18 themes.
  • Layout templates you can save and recall, plus auto-resume after a restart or crash.

# check on it from anywhere
  • A web client for your phone or another machine: agent list, chat, approvals, and a live terminal mirror.
  • Layout mirroring, tmux style, keeps desktop and web in sync on cards, tabs, and the active tab.
  • Opt-in sharing with a bearer token and a QR dialog, off by default.
  • One bus, every client, so a card never doubles up no matter where you open it.

# one quiet daemon, one bus, every client

The desktop app spawns and supervises the daemons. Sessions live in the daemon, so closing a window never kills an agent.

desktop client

Electron + React

The primary GUI. Spawns the daemons, renders the panes, forwards events to every client.

session daemon

claudemon [rust]

Owns the sessions and PTYs, takes hook events, runs the state machine, streams transcripts and git.

control plane

hub [go]

The event bus, supervisor, capability router, plugin system, and MCP facade.

extend it without forking
plugins drop in a manifest, get a supervised sidecar + its own panes + your theme
mcp the hub exposes the fleet as MCP tools so a supervisor agent can drive it
wks-tui a rust terminal client over the same bus, with a ctrl-k palette + vim keys
Point it at a repo and spawn your first agent.

Install the deps, run it in dev, and you are watching a fleet in a couple of minutes.
